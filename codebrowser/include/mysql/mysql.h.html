486/*
487 This structure is used to define bind information, and
488 internally by the client library.
489 Public members with their descriptions are listed below
490 (conventionally `On input' refers to the binds given to
491 mysql_stmt_bind_param, `On output' refers to the binds given
492 to mysql_stmt_bind_result):
493
494 buffer_type - One of the MYSQL_* types, used to describe
495 the host language type of buffer.
496 On output: if column type is different from
497 buffer_type, column value is automatically converted
498 to buffer_type before it is stored in the buffer.
499 buffer - On input: points to the buffer with input data.
500 On output: points to the buffer capable to store
501 output data.
502 The type of memory pointed by buffer must correspond
503 to buffer_type. See the correspondence table in
504 the comment to mysql_stmt_bind_param.
505
506 The two above members are mandatory for any kind of bind.
507
508 buffer_length - the length of the buffer. You don't have to set
509 it for any fixed length buffer: float, double,
510 int, etc. It must be set however for variable-length
511 types, such as BLOBs or STRINGs.
512
513 length - On input: in case when lengths of input values
514 are different for each execute, you can set this to
515 point at a variable containining value length. This
516 way the value length can be different in each execute.
517 If length is not NULL, buffer_length is not used.
518 Note, length can even point at buffer_length if
519 you keep bind structures around while fetching:
520 this way you can change buffer_length before
521 each execution, everything will work ok.
522 On output: if length is set, mysql_stmt_fetch will
523 write column length into it.
524
525 is_null - On input: points to a boolean variable that should
526 be set to TRUE for NULL values.
527 This member is useful only if your data may be
528 NULL in some but not all cases.
529 If your data is never NULL, is_null should be set to 0.
530 If your data is always NULL, set buffer_type
531 to MYSQL_TYPE_NULL, and is_null will not be used.
532
533 is_unsigned - On input: used to signify that values provided for one
534 of numeric types are unsigned.
535 On output describes signedness of the output buffer.
536 If, taking into account is_unsigned flag, column data
537 is out of range of the output buffer, data for this column
538 is regarded truncated. Note that this has no correspondence
539 to the sign of result set column, if you need to find it out
540 use mysql_stmt_result_metadata.
541 error - where to write a truncation error if it is present.
542 possible error value is:
543 0 no truncation
544 1 value is out of range or buffer is too small
545
546 Please note that MYSQL_BIND also has internals members.
547*/
